## 复测重点

1. 默认采样间隔均为 1000ms

PASS。

- `FrameScopeConfigStoreTests.exe` 覆盖默认 `ProcessSampleIntervalMs=1000`、`SlowSampleIntervalMs=1000`、`CpuTelemetry.PerCoreSampleIntervalMs=1000`、`CpuTelemetry.PerCoreVoltageSampleIntervalMs=1000`。
- `mockPreview.test.ts` 在前端 verify 中覆盖 mock/default preview 的 1000ms 默认值。
- synthetic monitor-session 的 `status.json` / `summary.json` 均记录：`SampleIntervalMs=1000`、`ProcessSampleIntervalMs=1000`、`SlowSampleIntervalMs=1000`、`CpuCoreSampleIntervalMs=1000`、`CpuVoltageSampleIntervalMs=1000`。

说明：当前真实 `framescope-config.json` 里已有旧 target 保留 `ProcessSampleIntervalMs=100`，WebView2 smoke 读取到的是这个本机既有配置；默认/新建配置路径仍由测试证明为 1000ms，旧显式配置被保留是兼容行为。

2. Settings UI 修改间隔，保存后 config/bridge/monitor-session 使用新值

PASS。

- `uiInteractionContract.test.ts` 覆盖 Settings 中存在四个采样间隔控件：后台进程、系统慢采样、CPU 核心频率、CPU 核心电压。
- `FrameScopeWebBridgeTests.exe` 覆盖 `config.save` 保存并返回 `PerCoreSampleIntervalMs=1500`、`PerCoreVoltageSampleIntervalMs=1750`，同时保存 CPU frequency/voltage toggle 和 provider。
- `FrameScopeConfigStoreTests.exe` 覆盖默认值、上下限 clamp、保存/加载 roundtrip。
- 补充 monitor-session 以非默认值运行：`ProcessSampleIntervalMs=1500`、`SlowSampleIntervalMs=2000`、`CpuCoreSampleIntervalMs=1500`、`CpuVoltageSampleIntervalMs=1750`。`custom-interval-monitor-session-evidence.json` 中 `statusIntervals`、`summaryIntervals`、telemetry status 全部记录了这些新值。
